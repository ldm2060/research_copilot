# Skills Overview

Every skill under `self/skills/` lives in its own subdirectory with a `SKILL.md`. Claude Code matches by the frontmatter `description` semantics (and a sibling `skill.json` for 2.1.142+ discovery — autogenerated). Users can also trigger explicitly via `/skill-name`.

## 🚀 One-shot install

Run once at the repo root before first use:

```bash
python self/install.py
```

The installer configures MCP servers, registers hooks, installs Python deps, and regenerates `skill.json` for every skill. See [`self/README.md`](README.md).

To refresh `skill.json` files alone (after editing SKILL.md frontmatter):

```bash
bash self/scripts/generate-skill-json.sh
```

## 🔌 MCP servers

Configured by `python self/install.py`. All servers live under `self/mcp/servers/`; the config file is written to the project-level `.mcp.json` by the installer.

| Server | Location | Description |
|---|---|---|
| **ai-scientist** | `self/mcp/servers/ai-scientist/` | 🧪 Non-model AI-Scientist ops: runtime check, experiment directory browsing |
| **arxiv-search** | `self/mcp/servers/arxiv-search/` | 🔍 arXiv search (keyword / author / arXiv ID), 3-second rate-limit + 429 retry |
| **arxivsub-search** | `self/mcp/servers/arxivsub-search/` | 🔍 arXiv + top-venue joint search, requires `ARXIVSUB_SKILL_KEY` |
| **dblp-bib** | `self/mcp/servers/dblp-bib/` | 📚 DBLP BibTeX query (standard entries, ready to paste into `references.bib`), 1.5-second rate-limit |
| **google-scholar** | `self/mcp/servers/google-scholar/` | 🔎 Scholar metadata / citation formats (fallback when arXiv has no hit) |
| **pdf-text** | `self/mcp/servers/pdf-text/` | 📄 Per-page PDF text extraction (`pdfplumber` preferred / `PyPDF2` fallback) |
